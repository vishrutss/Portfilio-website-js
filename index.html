Chess AI

A Chess AI project where the user can play Chess against an AI. Implemented the Negamax algorithm to improve AI’s decision making and also implemented Alpha-Beta pruning to improve the AI’s efficiency in finding the best moves.

I have Implemented a depth system for the AI where I can designate a value to Depth depending on how much processing I want the AI to do. For example if the depth is set at 1 the AI looks 1 move ahead, at 2 the AI looks 2 moves ahead and at 3 it looks 3 moves ahead. As the depth increases so does the time taken to make decisions by the AI.

The basic principle used is that the AI generates all the possible moves available to it and also generates all possible moves that the opponent can make. More the value of Depth the more this process is repeated. Since it can result in a massive number of moves for the AI to consider I implemented Alpha-Beta pruning. By using this algorithm I was able to reduce the load on the AI significantly. By implementing Alpha-Beta pruning the AI disregards moves that don’t benefit it and it disregards moves if it has already found the optimal move.

For example let's take a scenario where the AI has an option to capture the Queen or it has an option to capture a pawn which might allow it to capture the opponents Bishop on the next move. With the help of Alpha Beta pruning the AI disregards the latter move tree where it is able to capture 2 pieces but their value combined does not match the value of capturing the Queen so the AI takes less time deciding to go ahead and capture the Queen.

Tech: Python, PyCharm, PyGame libraries.

Product Key Activation using Asymmetric Cryptography

Designed an application that acted as a product activation portal. The product keys were generated using the user’s mac address. The program implements asymmetric cryptography to encrypt and decrypt the keys being sent between the user and the server.

At first when the user wants to buy the product key they provide the application with the necessary details. They application then saves the user’s MAC address and then encrypts the data using RSA. The server then decrypts this data after receiving it. It then uses the collatz conjecture method to generate a unique Product key for the user.

The process of using the collatz conjecture is that the server applies it to each digit of the MAC address that it receives. This results in a very large number which cannout be used as a Product key. To reduce the size of this number I used the RSA's encryption formula with preset values. The resulting number is then sent to the user's email address.

The application verifies the product key by generating the key locally and makes sure the key entered is accurate. An improvement on this process would be to reverse the process done by the server i.e, decrypting using RSA and then reversing the collatz conjecture calculations to arrive at a MAC address. The application can then check whether the user's MAC address matches the MAC address that it decrypts.

Driver Drowsiness Detection

Designed a program to detect the drowsiness of a driver using Viola Jones image detection algorithm with the selection of the eyes and mouth. There are 2 seperate functions that check the eyes and the mouth. Eye detection is done by looking for 2 dark spots(the eyes) split by a bright spot(the nose) since eyes look darker than the nose and cheeks on video.

Here the program checks if the eyes are open or closing. It checks the amount of white pixels present whithin the area it perceives as the eyes. If there are a considerable amount of white pixels it means that the eyes are open otherwise the eyes are closed.

Mouth detection is done similar to that of the eyes. The program looks for 2 dark spots(the eyes and the mouth) and a bridge in between(the nose and cheeks). The program checks if the mouth is open by checking if the lips are parted and there are a considerable amount of white pixels between the lips(the teeth). If the lips are parted this means that the driver is yawning/talking.

These 2 programs are then combined to create a probability function to avoid wrong detection. The probability values are as follows

Mouth Closing Split
Eyes
Open 100% 84%
Close 50% 15%

Once the probability has been determined a random number is generated and the application checks in what range the number falls. It then gives the final verdict wheter the driver is drowsy or awake. Since the program runs a check constantly every second there is a very low chance of the application performing a wrong prediction constantly for a long period of time.
